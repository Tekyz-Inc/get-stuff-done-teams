Key Findings

HIGHSecurity
SEC-H01: gsd-t-update-check.js

`latest` comes from `JSON.parse(d).version` in an inline HTTP response handler. No semver validation before string concatenation. A MITM attacker or compromised npm registry could set `version` to `"1.0.0; rm -rf ~"` and achieve shell command injection. execSync is a shell call, so semicolons/pipes execute.

RecommendationValidate with `/^\d+\.\d+\.\d+(-[a-zA-Z0-9.]+)?$/` before use. Use `execFileSync('npm', ['install', '-g', '@tekyzinc/gsd-t@' + latest])`.
HIGHSecurity
SEC-H02: gsd-t-update-check.js

Low immediate risk (no user input). Inconsistent with codebase standard of execFileSync with array args. If refactored with parameterization, injection risk emerges.

Recommendation`execFileSync('gsd-t', ['update-all'], { ... })`
HIGHSecurity
SEC-H03: scan-export.js

`htmlPath` derives from `opts.projectRoot` which may be caller-controlled. A path containing special shell characters can achieve injection. Double-quote wrapping provides minor protection but fails against embedded double-quotes.

Recommendation`execFileSync('pandoc', [htmlPath, '-o', outputPath, '--from=html'], { ... })`
MEDIUMSecurity
SEC-M01: scan-renderer.js

tmpIn/tmpOut come from `os.tmpdir() + Date.now()`. Risk is low on most platforms since os.tmpdir() returns a system path. On systems where TMPDIR is user-controlled, risk increases.

Recommendation`execFileSync('mmdc', ['-i', tmpIn, '-o', tmpOut, '-t', 'dark', '--quiet'], { ... })`
MEDIUMSecurity
SEC-M02: gsd-t-tools.js

`value` parameter written directly to progress.md without newline sanitization. A value of `"x\n## New Section\n..."` injects a heading into the file.

RecommendationAdd `value = String(value).replace(/[\r\n]/g, ' ')` before the replace operation.
MEDIUMSecurity
SEC-M03: scan-export.js

`cmd` parameter is currently only called with hardcoded 'pandoc'. Function accepts arbitrary strings. Future callers with dynamic tool names could trigger injection via the `'where "' + cmd + '"'` pattern.

RecommendationValidate `cmd` against `/^[a-zA-Z0-9_-]+$/` before use, or use execFileSync with array args.
