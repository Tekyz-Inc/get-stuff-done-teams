# Domain: design-brief

## Responsibility
Design brief artifact for UI-heavy projects, generated during partition/plan, injected into execute subagents for UI tasks.

## Files Owned
- None (no new JS modules — design brief is a contract-like .md artifact generated by partition)

## Files Touched
- `commands/gsd-t-partition.md` — design brief detection and generation step
- `commands/gsd-t-plan.md` — reference design brief in UI task descriptions
- `commands/gsd-t-setup.md` — design brief generation for existing projects

## Constraints
- Design brief stored at `.gsd-t/contracts/design-brief.md`
- Only generated when UI/frontend domains detected (React, Vue, Svelte, Flutter, CSS files)
- Derived from existing Tailwind config, theme files, or quality persona if no design system exists
- Non-UI tasks skip design brief injection
- Projects without UI domains skip entirely (no overhead)
