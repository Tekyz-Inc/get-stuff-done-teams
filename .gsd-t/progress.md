## Current Milestone

**M43 Token Attribution & Headless-Default Inversion** ‚Äî DEFINED (6 domains). Target: v3.17.10.

**Goal**: Make every token, in every turn, in every command, attributable to a specific tool / command / domain ‚Äî and use that visibility to flip the framework default so long-running work runs detached unless the user explicitly asks for in-session. Together: A measures the cost, B reduces it.

**Rationale**: Bee-poc compaction pathology (agent `b94ca63f`, 3 compactions in one workday with the interval collapsing 51min ‚Üí 11min) showed that Bash-heavy + Read-heavy in-session workloads need both visibility AND default-headless to avoid silent `/compact`. The two halves ship together so the inversion can be validated against attributed data immediately rather than guessed at.

**Scope ‚Äî Part A: Universal Token Attribution**
- **D1 per-turn in-session usage capture** ‚Äî Capture token usage for every turn of an in-session Claude Code session (not just headless `claude -p` spawns, which M41 already covers). Hook-based if Claude Code exposes `usage` on `Stop` / `SessionEnd`; otherwise extend the M42 transcript tee to interactive sessions and parse the same stream-json `result` frame the M40 D4 aggregator already understands. Output: per-turn rows into `.gsd-t/metrics/token-usage.jsonl` with `turn_id`, `session_id`, `command`, `model`, `usage{...}`.
- **D2 per-tool attribution** ‚Äî `bin/gsd-t-tool-attribution.cjs` joins the events stream (`.gsd-t/events/*.jsonl`, which carries every tool-call request/response with `turn_id`) against the per-turn usage JSONL from D1, then attributes the turn's usage across the turn's tool_calls by output-byte ratio (a tool whose `tool_result` was 80% of the turn's tool output gets credited 80% of the turn's tokens). Ships `gsd-t tool-cost --group-by tool|command|domain [--since YYYY-MM-DD] [--milestone Mxx] [--format table|json]`.
- **D3 sink unification + backfill** ‚Äî Define one canonical token-row schema (extending M40 D4 schema v1) and route every producer through it: M36 heartbeat, M40 aggregator, D1 hook, D2 joiner. `.gsd-t/token-log.md` becomes a *regenerated view* (`gsd-t tokens --regenerate-log`), not a hand-maintained markdown table. Backfill the existing 64 `Tokens=0` rows from `.gsd-t/events/*.jsonl` + `.gsd-t/headless-*.log` (M41 D3 backfiller existed but was never run against the historical log).

**Scope ‚Äî Part B: Default-Headless Inversion**
- **D4 flip command-file defaults** ‚Äî Invert the spawn-mode logic in ~14 command files (`execute`, `wave`, `integrate`, `quick`, `debug`, `verify`, `complete-milestone`, `test-sync`, `scan`, `gap-analysis`, `populate`, `feature`, `project`, `partition`) from "spawn headless only when over context band" to "spawn headless unless `--in-session` is explicitly passed OR session-context is below a low-water mark (~15%)". The router (`/gsd`) gets the inverse hint: "this is a quick exploratory question ‚Äî running in-session unless you pass `--headless`."
- **D5 compaction-pressure circuit breaker** ‚Äî Extend `bin/runway-estimator.cjs` (M35) with a "compaction pressure" signal that fires when the recent-turn usage trajectory predicts the next `/compact` within N turns. When tripped, in-session commands refuse to start and route to headless instead, with a clear operator message. Closes the loophole where the per-call meter looks fine but turn-over-turn growth is the real killer.
- **D6 transcript viewer as primary surface** ‚Äî Promote the M42 `:7433` transcript viewer from "useful for unattended runs" to "default surface for any non-trivial command." Router prints a dashboard URL hint at every spawn; `--in-session` mode prints "view live at http://127.0.0.1:7433/transcript/{spawn-id}" so users get the same scroll-back UX whether the command runs in-session or detached.

**Out of scope (deferred)**: Intervene / SIGSTOP-inject (already deferred from M42). Migrating away from `usage`-bearing streams if Claude Code stops emitting them (we depend on the contract ‚Äî if it breaks, that's a separate forensic milestone).

**Contracts touched**:
- `metrics-schema-contract.md` v1 ‚Üí v2 (adds `turn_id`, `session_id`, `tool_attribution[]`, `compaction_pressure`)
- `stream-json-sink-contract.md` v1.1.0 ‚Üí v1.2.0 (formalize the in-session hook entry-point alongside the existing headless-spawn entry-point)
- New: `tool-attribution-contract.md` (output-byte ratio algorithm, tie-breaker rules, missing-data handling)
- `headless-default-contract.md` v1.0.0 ‚Üí v2.0.0 (flip default; document `--in-session` opt-out + low-water mark)

**Success criteria**:
1. Every spawn (in-session OR headless) writes a per-turn row to `.gsd-t/metrics/token-usage.jsonl` with usage parsed (never `0`, never `N/A` ‚Äî `‚Äî` only when the upstream envelope itself is missing).
2. `gsd-t tool-cost --group-by tool` shows non-zero attribution for at least Bash, Read, Edit, Grep, Task across the last 30 days of repo activity (after backfill).
3. `.gsd-t/token-log.md` is fully regeneratable from `.gsd-t/metrics/token-usage.jsonl` (delete + regenerate produces a byte-identical file modulo timestamp ordering).
4. Default invocation of `/gsd-t-execute` / `/gsd-t-wave` / `/gsd-t-quick` (no flags) spawns headless; `--in-session` round-trips correctly through the router.
5. The compaction-pressure circuit breaker fires correctly on a synthetic high-growth fixture and routes to headless without silent `/compact`.

**Assessment**: COMPLEX (6 domains across 2 themes). Recommended flow: `partition` ‚Üí `discuss` (for D1 hook-vs-tee branch + D5 compaction-pressure heuristic) ‚Üí `plan` ‚Üí `execute` (waves) ‚Üí `integrate` ‚Üí `verify` ‚Üí `complete-milestone` ‚Üí tag v3.17.10.
