## Current Milestone

**M43 Token Attribution & Always-Headless Inversion** — DEFINED (6 domains). Target: v3.17.10.

**Goal**: Make every token attributable to a specific tool / command / domain, and lock the framework to a single rule: **the in-session channel is reserved for human↔Claude dialog. All tool-using work spawns. The visualizer is the watching surface.** No flags, no thresholds, no opt-outs — there is no "in-session mode" for commands to enter.

**Rationale (revised 2026-04-21)**: The original framing ("headless when pressure is high, in-session otherwise") is wrong. A circuit breaker only makes sense when the default is "in-session, sometimes headless." Once the default is "always headless, dialog-only stays in-session," there is nothing to trip — the spawn already happened. The dialog channel exists for *thinking together*; the visualizer (`:7433`) exists for *watching the doing*. Conflating them is what produced the bee-poc compaction pathology (agent `b94ca63f`, 3 compactions in one workday). The fix isn't smarter routing — it's a clean separation of channels.

**Scope — Part A: Universal Token Attribution**
- **D1 per-turn in-session usage capture** — Capture token usage for every turn of the dialog channel (the only thing still running in-session under this model). Hook-based if Claude Code exposes `usage` on `Stop` / `SessionEnd` (probe shipped `f94857b`, branch decision pending); otherwise extend the M42 transcript tee to interactive sessions and parse the same stream-json `result` frame the M40 D4 aggregator already understands. Output: per-turn rows into `.gsd-t/metrics/token-usage.jsonl` with `turn_id`, `session_id`, `command`, `model`, `usage{...}`. Headless spawns are already covered by M41.
- **D2 per-tool attribution** — `bin/gsd-t-tool-attribution.cjs` joins the events stream (`.gsd-t/events/*.jsonl`, which carries every tool-call request/response with `turn_id`) against the per-turn usage JSONL from D1, then attributes the turn's usage across the turn's tool_calls by output-byte ratio. Ships `gsd-t tool-cost --group-by tool|command|domain [--since YYYY-MM-DD] [--milestone Mxx] [--format table|json]`. Under the always-headless model this matters *more*, not less, because spawned work is now the dominant cost and operators need to slice it.
- **D3 sink unification + backfill** — Define one canonical token-row schema (extending M40 D4 schema v1) and route every producer through it: M36 heartbeat, M40 aggregator, D1 dialog capture, D2 joiner. `.gsd-t/token-log.md` becomes a *regenerated view* (`gsd-t tokens --regenerate-log`), not a hand-maintained markdown table. Backfill the existing 64 `Tokens=0` rows from `.gsd-t/events/*.jsonl` + `.gsd-t/headless-*.log` (M41 D3 backfiller existed but was never run against the historical log).

**Scope — Part B: Always-Headless Inversion (revised — channel separation, not threshold flip)**
- **D4 always-headless command spawn (no flag, no threshold)** — Strip the "headless when over context band" logic from ~14 command files (`execute`, `wave`, `integrate`, `quick`, `debug`, `verify`, `complete-milestone`, `test-sync`, `scan`, `gap-analysis`, `populate`, `feature`, `project`, `partition`). Every command spawns. Period. **No `--in-session` opt-out, no `--headless` flag, no low-water mark, no context-meter-driven branching at spawn time.** The only thing that runs in the dialog channel is the `/gsd` router itself, which always spawns the actual command. Removes ~3 layers of conditional routing logic and the cognitive load of "which mode is this command in right now."
- **D5 dialog-channel meter (router-only, replaces the deleted compaction-pressure circuit breaker)** — Originally scoped as a turn-over-turn circuit breaker that would refuse to start in-session commands when growth predicted `/compact`. Under the new channel model there are no in-session commands to refuse. **D5 collapses to a much smaller utility**: a turn-over-turn growth meter on the router itself (since the router is what stays in-session). When dialog growth predicts `/compact` within N turns, surface a one-line warning so the user can `/clear` or wrap the conversation deliberately. Pure read/warn — never refuses, never reroutes (there's nothing to reroute to). Lives in `bin/runway-estimator.cjs` as a small extension, not a new module.
- **D6 transcript viewer as the primary surface (not a hint)** — Promote the M42 `:7433` transcript viewer from "useful for unattended runs" / "URL hint alongside in-session option" to **the** surface. Every spawn prints `view at http://127.0.0.1:7433/transcript/{spawn-id}` because that's where the work lives now. The dashboard auto-launches on first command spawn if not already running (port resolved via the project-hashed default from the multi-project isolation quick `df34eb2`). No `--in-session` print path because there is no `--in-session` mode.

**Tradeoff acknowledged**: Spawning even tiny work (a single Read, one Bash) has overhead — orchestrator startup, transcript file, dashboard registration. Worth it for clean channel separation and uniform observability. If overhead bites, the answer is to make the spawn faster (M40-style fast spawn), not to make a threshold smarter.

**Out of scope (deferred)**:
- Intervene / SIGSTOP-inject (already deferred from M42).
- Migrating away from `usage`-bearing streams if Claude Code stops emitting them (separate forensic milestone if the contract breaks).
- Auto-`/clear` on dialog meter trip — D5 only warns; the user decides when to wrap the dialog.

**Contracts touched**:
- `metrics-schema-contract.md` v1 → v2 (adds `turn_id`, `session_id`, `tool_attribution[]`; **removes** `compaction_pressure` from the original v2 sketch — D5 collapses, no schema field needed)

- `headless-default-contract.md` v1.0.0 → v2.0.0 (always-headless rule; **delete** `--in-session` opt-out + low-water mark from the v1 spec; document the channel-separation invariant: dialog = router only, everything else spawns)

**Success criteria**:
1. Every spawn writes a per-turn row to `.gsd-t/metrics/token-usage.jsonl` with usage parsed (never `0`, never `N/A` — `—` only when the upstream envelope itself is missing).
2. `gsd-t tool-cost --group-by tool` shows non-zero attribution for at least Bash, Read, Edit, Grep, Task across the last 30 days of repo activity (after backfill).
3. `.gsd-t/token-log.md` is fully regeneratable from `.gsd-t/metrics/token-usage.jsonl` (delete + regenerate produces a byte-identical file modulo timestamp ordering).
4. **No command file accepts `--in-session` or `--headless` flags.** Default invocation of any of the 14 listed commands spawns. Grep for `--in-session` in `commands/*.md` returns zero matches after D4.
5. Dashboard URL is printed at every spawn; visiting it shows the live transcript without operator action beyond clicking the link.
6. D5 dialog meter fires a one-line warning on a synthetic high-growth fixture in the router context; never refuses, never reroutes.

**Assessment**: MEDIUM (was COMPLEX — D5 collapse and D4 simplification each removed roughly half their original scope).
