### M39 — Fast Unattended + Universal Watch-Progress Tree

**Status**: PLANNED
**Target version**: 3.13.10 (minor bump — new user-visible capabilities: parallel unattended + watch-progress tree)
**Defined**: 2026-04-17

**Goal**: Close the 3–5× speed gap between unattended and in-session execution by teaching the unattended worker to use Team Mode for multi-domain waves (cap 15 parallel subagents, intra-wave only). Ship a universal task-list progress view that renders under every `--watch` surface, showing the full workflow tree from requirements → partition → milestone(s) → plan/execute/verify/complete. Close supervisor→worker handoff gap below the 5-min Anthropic prompt cache TTL.

**Why this milestone exists**:
- bee-poc's current unattended run (pid 69481) has been grinding on M7 iter=1 for 45+ min on v3.12.13. The same milestone in-session with Team Mode finishes in ~10–15 min. Root cause: `bin/gsd-t-unattended.cjs::_spawnWorker` (~line 1135) prompt doesn't teach the worker to use Team Mode — it executes tasks sequentially even when the wave has N independent domains.
- Watch-output is minimal. No mental model of progress for the user. In-session shows a task list when there is one; unattended shows nothing.
- Supervisor→worker handoff today crosses the 5-minute Anthropic prompt cache TTL, so every iter pays a cold-cache penalty.

**Success criteria**:
1. **Parallel-within-iter works.** Unattended worker, handed a wave with multiple independent domains/tasks, spawns up to 15 parallel Task subagents (one per domain), waits for all, then advances. Intra-wave parallel, inter-wave sequential (wave dependency graph preserved).
2. **Universal watch-progress tree renders under every `--watch`.** A unified task-list view: collapsed at outer levels, expanded at the current inner level. State-file-driven (`.gsd-t/.watch-state/{agent_id}.json`) — each subagent writes its own state; watcher rebuilds the tree and renders under the existing banner (banner preserved intact).
3. **Cache-warm pacing.** Supervisor→worker handoff stays under the 5-min prompt cache TTL (target 270s worker timeout + immediate re-spawn, not 300s+ gap).
4. **bee-poc speedup verified.** After M39 ships and bee-poc's supervisor is relaunched on v3.13.10, the next iter completes 3–5× faster than the v3.12.13 baseline, and the task list visibly progresses.

**Scope — included**:
- D2 `progress-watch`: `.gsd-t/.watch-state/{agent_id}.json` state-file schema, `bin/watch-progress.js` tree builder + renderer, command-file shims to write state at step start/end, integration into `--watch` output (append below banner, keep banner intact).
- D3 `parallel-exec`: `_spawnWorker` prompt teaches Team Mode for multi-domain waves with cap 15; sequential fallback when wave has only 1 domain; `unattended-supervisor-contract.md` update.
- D4 `cache-warm-pacing`: supervisor pacing logic (worker timeout tuning, re-spawn latency budget), keeps each iter within the 5-min cache window.

**Scope — explicitly NOT included**:
- Decision Log trim — already shipped in v3.12.15 (what was M39 D1 is now dropped).
- Cross-wave parallelism — wave boundaries stay sequential (contract-safety).
- Rewriting supervisor CLI surface — keep existing `/gsd-t-unattended`, `/gsd-t-unattended-stop`, `/gsd-t-resume` verbs.
- Changing `event-schema-contract.md` event types — reuse existing `phase_transition` / `subagent_spawn` / `subagent_complete`.

**Complexity**: Medium (3 domains). Standard partition → plan → execute → test-sync → verify → complete-milestone chain. Chain runs without user confirmation between phases; only pause for (a) destructive actions (`npm publish` at close), (b) unresolvable blockers, (c) Red Team FAIL after 2 fix cycles.

**Release**: Closes with `npm publish` as v3.13.10. Then `npm install -g @tekyzinc/gsd-t@3.13.10 --force`, `gsd-t update-all`, final verification that the task list renders in a test `--watch` run.
